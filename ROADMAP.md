## v1.5 (2월 말) - policy.yml 베타

> **목표**: 팀별 정책 정의 기능 시작

### 기능

| 작업               | 상태        | 설명                 |
| ------------------ | ----------- | -------------------- |
| clouvel.policy.yml | 💡 설계     | 선언적 정책 파일     |
| 팀별 규칙 적용     | 💡 설계     | policy.yml 기반 검증 |
| 예외 승인 로그     | 💡 아이디어 | 예외 사유/기간 기록  |

### policy.yml 예시

```yaml
version: "1.0"
name: "my-team-policy"

require_docs:
  - PRD.md
  - ARCH.md

require_sections:
  PRD.md: [scope, non_goals, acceptance]

quality_gates:
  min_tests: 8

block_on: [missing_healthz, no_acceptance_criteria]
warn_on: [no_arch_doc, low_test_count]
```

### 성공 지표

| 지표               | 목표 |
| ------------------ | ---- |
| policy.yml 사용 팀 | 5+   |
| Free → Pro 전환    | 5건+ |

### 기능

| 작업             | 상태        | 설명                |
| ---------------- | ----------- | ------------------- |
| Evidence Pack v1 | 💡 설계     | 검증 결과 자동 기록 |

| 감사 대응 기초   | 💡 아이디어 | 기록 조회/내보내기  |

### Evidence Pack 구조

```
.clouvel/evidence/
├── 2026-03-01_12-00-00/
│   ├── result.json          # 검증 결과
│   ├── policy.yml           # 적용된 정책
│   ├── input_hashes.json    # 입력 파일 해시
│   └── change_summary.md    # 변경 요약
```

## v3.0 (Q3~Q4) - B2B급 방어선

> **목표**: 에이전트가 강해져도 Clouvel이 필수 인프라로 남기

### 기능

| 작업            | 상태        | 설명                    |
| --------------- | ----------- | ----------------------- |
| Safety Layer    | 💡 아이디어 | 에이전트 위험 행동 차단 |
| Eval Harness    | 💡 아이디어 | 정책 효과 증명          |
| Compliance Pack | 💡 아이디어 | 규제/감사 대응 자동화   |

### Safety Layer 위험 분류

| 레벨     | 행동            | 기본 동작    |
| -------- | --------------- | ------------ |
| CRITICAL | 대량 삭제 (10+) | BLOCK + 알림 |
| CRITICAL | 시크릿 접근     | Human 확인   |
| HIGH     | 대규모 리팩터링 | WARN + 로그  |
