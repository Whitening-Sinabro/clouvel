문제

Claude Code는 빠릅니다. 하지만 메모리 없는 속도는 혼란을 만듭니다.

1

AI가 어제 고친 버그를 다시 만듦

세션 간 에러 메모리 없음. 같은 실수, 다른 날.

2

아무도 리뷰 없이 배포

두 번째 눈 없음. 맹점이 프로덕션으로 직행.

3

"왜 이렇게 했더라?"

컨텍스트 리셋되면 결정 사라짐. 다시 파악하느라 시간 낭비.

4

새 세션 = 같은 실수

AI가 매번 제로에서 시작. 학습 없음, 개선 없음.

Clouvel이 하는 일

세 가지. 전부 자동.

1. 에러 메모리

실수에서 배우는 AI.

경고: 이 에러가 전에 발생한 적 있습니다.

근본 원인: DB 쿼리 결과 null 체크 누락

예방: 쿼리 결과 항상 검증

(메모리 #7 - 3번 방지됨)

2. 스펙 게이트

AI가 코딩하기 전에 생각.

You: "로그인 만들어"

차단 - PRD 없음.

You: *PRD 작성*

PASS - 코딩 진행 가능.

clouvel install

# py -m clouvel install      (Windows)
# python3 -m clouvel install (Mac/Linux)

# 3. 코딩 시작

claude

자주 묻는 질문

Clouvel이 정확히 뭐 하는 건가요?

Clouvel은 Claude Code 옆에서 실행되는 MCP 서버입니다. 모든 에러를 근본 원인 분석과 함께 기록하고, 과거 실수를 반복하기 전에 경고하고, 스펙 문서 없이 코딩을 차단합니다.

CLAUDE.md 쓰면 안 되나요?

CLAUDE.md는 직접 작성하는 정적 규칙입니다. Clouvel은 실제 에러와 결정에서 자동으로 학습합니다. 아직 만나지 못한 버그에 대한 CLAUDE.md 규칙은 쓸 수 없습니다.

오프라인에서 작동하나요?

네. Pro 라이선스 활성화에만 인터넷 한 번 필요. 이후 모든 기능 완전 오프라인. 모든 데이터는 로컬에 저장됩니다.

코드나 프로젝트 파일을 건드리나요?

아닙니다. Clouvel은 MCP 서버라서 Claude Code에 도구만 제공하고 소스 코드를 절대 수정하지 않습니다. 데이터는 .claude/ 폴더에 저장됩니다.

Free가 진짜 쓸만한가요?

네. 작동하는 진짜 10개 도구. 스펙 게이팅, 에러 트래킹(최근 5개), 컨텍스트 저장/로드, 퀵 체크 등. Pro는 전체 기록, 패턴 학습, 크로스 프로젝트 메모리, 매니저 리뷰를 추가합니다.

서버로 어떤 데이터가 전송되나요?

딱 두 가지: 라이선스 키와 머신 ID(하드웨어 정보의 SHA-256 해시 — 원본 정보는 절대 전송되지 않음). Pro 활성화 시 한 번만 전송됩니다. 이후에는 완전 오프라인. 코드, 문서, 에러 기록은 절대 로컬을 벗어나지 않습니다.

컴퓨터를 바꾸면 어떻게 하나요?

위 문의 양식 또는 GitHub Issue에 라이선스 키를 보내주세요. 보통 24~48시간 내에 머신 ID를 초기화해드립니다.
