한 번 설치하면 어디서든 같은 정책

AI 코딩, 왜 자꾸 어긋나는가?

1

AI가 요구사항 일부를 임의로 생략해서 구현이 어긋남

"로그인 기능 만들어줘" → 비밀번호 재설정? 소셜 로그인? 알아서 생략됨

2

중간에 코딩부터 시작해버려서 설계/검증이 뒤늦음

아키텍처 고민 없이 바로 코드 → 나중에 전체 리팩토링

3

결과가 매번 달라져서 디버깅 시간이 폭발

같은 요청인데 어제랑 오늘 결과가 다름 → 2-3일 디버깅 지옥

그래서 PRD를 '선택'이 아니라 '게이트'로 만든다.

How it works

3단계로 끝

1

PRD 제공

없으면 진행 불가. 문서부터 작성해야 시작됨.

2

Gate 체크

필수 섹션, 형식, 결정론 검사를 통과해야 함.

3

코딩 허용

통과한 경우에만 코딩 시작. 체계적 개발.

진짜 막히는지 보세요

3단계 품질 게이트: BLOCK → WARN → PASS

PRD 없음 BLOCK

$ claude "로그인 만들어줘"

BLOCK: 코딩 금지

PRD 없음 → 문서 먼저

PRD 불완전 WARN

$ claude "로그인 만들어줘"

PASS | WARN 2개

권장 없음: 아키텍처, 테스트

PRD 완전 PASS

$ claude "로그인 만들어줘"

PASS | 코딩 시작 가능

필수: PRD ✓ | 테스트 3개

딱 필요한 것만

핵심 기능 4가지

PRD-First 강제

문서 없으면 코딩 차단. 요구사항 누락 원천 방지.

결정론 보장

동일 입력 = 동일 출력. 결과 일관성 확보.

누락 감지

요구사항 빠뜨리면 즉시 알림. 사후가 아닌 사전에.

컨텍스트 복구

세션 끊겨도 맥락 유지. 컴팩션 후에도 자동 복구.

Getting Started

환경에 맞게 3단계로 설치

pip install clouvel

# 2. MCP 등록 (한 줄로 끝)

clouvel install

# 플랫폼 자동 감지 + 글로벌 규칙 설정

# 3. Run

claude

# 코드 작성 요청 시 자동으로 can_code 검사

# 2. MCP 등록

clouvel install --platform desktop

# config.json 자동 수정

# 3. Run

FAQ

자주 묻는 질문

PRD 형식 강제함?

기본 템플릿을 제공하지만, 강제하지는 않습니다. 필수 섹션(목표, 범위, 요구사항)만 있으면 됩니다.

내 PRD 템플릿 써도 됨?

네. 기존 템플릿 그대로 사용 가능합니다. Clouvel은 문서 "존재 여부"를 체크하는 게이트입니다.

오프라인/로컬에서 되나?

라이선스 활성화 시 최초 1회 인터넷 연결이 필요합니다. 이후에는 오프라인에서도 사용 가능합니다.

기존 워크플로에 어떻게 끼움?

MCP 서버로 동작하므로, Claude Code/Desktop에 설정만 추가하면 됩니다. 기존 코드나 프로젝트 구조는 건드리지 않습니다.

왜 그냥 프롬프트로 해결 안 함?

프롬프트는 '요청'이고, 게이트는 '규칙'입니다. 요청은 무시될 수 있지만, 게이트는 통과 아니면 차단입니다. 규칙이 있어야 일관성이 생깁니다.
