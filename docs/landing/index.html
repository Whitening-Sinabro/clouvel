What is a PRD?

A spec document that AI reads before coding.
With a PRD, AI knows exactly "what to build" and helps you accurately.

Without PRD

"Build a login feature"

AI guesses and builds something. Password reset? Social login? Skipped.

With PRD

AI reads PRD and implements exactly

"Email login, password reset, Google OAuth" - all included. Nothing missed.

PRD = AI's work order. Clear instructions, clear results.

FAQ

Frequently Asked Questions

Does it force a specific PRD format?

We provide templates, but don't force them. Just need required sections (goals, scope, requirements).

Can I use my own PRD template?

Yes. Use your existing template as-is. Clouvel just gates on document "existence", not specific format.

Does it work offline/locally?

License activation needs internet once. After that, works fully offline.

How does it fit my existing workflow?

It's an MCP server - just add config to Claude Code/Desktop. Doesn't touch your code or project structure.

Why not just use a prompt or CLAUDE.md?

A prompt is a "request". A gate is a "rule". Requests can be ignored. Gates either pass or block. Rules create consistency.
