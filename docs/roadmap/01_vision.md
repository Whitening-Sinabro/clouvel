# 01. 비전 및 핵심 가설

> Clouvel의 존재 이유와 방향성
> **Updated**: 2026-02-09 (리포지셔닝 반영)

---

## 비전

### 단기 비전 (2026)

```
"AI makes it fast. Clouvel makes it right."
- AI가 놓치는 것을 기억하고, 같은 실수를 반복하지 않게
- Free to Act, Paid to Remember
```

### 장기 비전 (2028+)

```
"개발자의 외부 뇌 (External Brain)"
- 프로젝트/도구/세션을 넘어 축적되는 기억
- AI가 빨라질수록, "무엇을 기억하는가"가 차이를 만듦
- 플랫폼은 생성을 담당하고, Clouvel은 기억을 담당
```

### 비전 진화 경로

```
pre-v1.0: 스펙 없으면 코딩 차단 (enforcement) ← 완료
          ↓
v1.0:     AI 실수 기억 + 재발 방지 (memory) ← 현재
          ↓
v1.x:     크로스 프로젝트 기억 전이 (transfer)
          ↓
v2.0+:    크로스 AI 도구 기억 레이어 (universal)
```

### 포지셔닝 전환 배경 (2026-02-09)

```
문제: AI 코딩 도구(Claude, Cursor, Copilot)가 자체적으로
      문서/계획 기능을 추가하면서 "PRD gate" 가치가 commodity화

4개 AI (Claude, ChatGPT, Gemini, Grok) 컨센서스:
1. MCP 서버 비즈니스는 장기적으로 흡수됨 (4/4 동의)
2. "강제" 대신 "기억"이 플랫폼이 못하는 영역
3. Error/History Learning이 1순위 유료 기능
4. "Free to Act, Paid to Remember"가 최적 전환 구조

결론: Gate(차단) → Memory(기억)으로 핵심 가치 전환
기존 기능(can_code, meeting 등)은 유지하되 포지셔닝 변경
```

### 가설 3: AI 플랫폼은 "기억"을 구조적으로 못한다

| 경쟁자           | 하는 것                  | 구조적으로 못하는 것                |
| ---------------- | ------------------------ | ----------------------------------- |
| Claude Code      | Plan mode, CLAUDE.md     | 크로스세션 에러 기억, 프로젝트 간 전이 |
| Cursor           | .cursorrules, 프로젝트 메모리 | 크로스 도구 기억, 실패 패턴 학습     |
| Copilot          | 컨텍스트 인식            | 영구 의사결정 기록, 부정 지식        |
| Windsurf         | Cascade memories         | 프로젝트 간 기억 전이               |

**검증 상태**: 🔄 검증 중 (2026-02-09 리포지셔닝)

**의미**: 플랫폼은 "생성 속도"에 최적화. "실수 기억/반복 방지"는 외부 도구의 영역.
경쟁자들은 경쟁 관계라 서로의 기억을 공유하지 않음 (구조적 불가).

## 철학

### 하지 않는 것 (Never)

```
❌ 코드 생성 — AI IDE의 역할
❌ 코드 수정 — AI IDE의 역할
❌ 자동 구현 — 개발자 + AI IDE의 역할
❌ 기술 스택 강제 — 어떤 기술이든 OK
❌ 데이터 인질 — archived 데이터도 export 항상 가능
```

### 하는 것 (Always)

```
✅ Remember (기억)
   - AI 실수를 기록하고, 다음에 같은 패턴에서 경고
   - 의사결정 이력을 크로스세션으로 유지
   - "왜 이렇게 했는지" 출처를 추적

✅ Prevent (예방)
   - 과거 실패 패턴 기반 재발 방지
   - "지난번에 이거 터졌잖아" 자동 경고
   - Negative Knowledge: "이건 쓰지 마" 자동 주입

✅ Enforce (강제) — 기존 유지
   - 스펙 없이 코딩 시작하면 차단
   - PRD가 없으면 can_code = BLOCK

✅ Verify (검증) — 기존 유지
   - 스펙 완성도 체크
   - lint → test → build 파이프라인

✅ Guide (가이드) — 기존 유지
   - 8 C-Level 매니저 관점 피드백
   - 올바른 구조/패턴 제안
```

### 철학 요약

```
Clouvel = AI가 빠르게 만든 것을, 올바르게 만들었는지 기억하는 도구

"AI forgets. You pay the price."
AI는 잊는다. 그 대가는 당신이 치른다.

Clouvel은 AI가 잊는 것을 기억한다.
```

## 타겟 사용자

### 1차 타겟: Solo Dev + Vibe Coders (2026)

```
프로필:
- AI 코딩 도구 적극 사용 (Claude Code, Cursor)
- 프로젝트 2-3개 운영
- "같은 실수 반복"에 지침
- 세션 넘기면 컨텍스트 리셋에 좌절

페인포인트:
- "지난번에도 이거 터졌는데..." 반복
- AI가 만든 코드 수정에 시간 낭비
- 세션 바뀌면 맥락 다 잊어버림

Clouvel 가치:
- 기억이 쌓일수록 실수가 줄어듦
- "이거 지난번에 터졌잖아" 자동 경고
- 크로스세션 의사결정 기억
```

### 2차 타겟: 프리랜서 / 에이전시 (2027)

```
프로필:
- 여러 클라이언트 프로젝트 운영
- 프로젝트 간 학습 전이 필요
- 결정/변경에 대한 증빙 필요

페인포인트:
- 프로젝트 A에서 배운 걸 B에서 까먹음
- "왜 이렇게 만들었나?" 증명 필요
- 클라이언트 분쟁 시 근거 부족

Clouvel 가치:
- 크로스 프로젝트 기억 전이
- 의사결정 감사 로그 (증빙)
- Source Tagging (출처 추적)
```

### 3차 타겟: 소규모 팀 (2028+)

```
프로필:
- 2-10명 개발팀, AI 도구 사용 중
- 팀원 간 일관성 필요

Clouvel 가치:
- 팀 공유 Knowledge Base
- 팀 수준 Regression Memory
- 공유 Negative Knowledge Vault
```
