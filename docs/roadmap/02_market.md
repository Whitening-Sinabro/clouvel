## Clouvel의 기회

### 핵심 인사이트

```
"84%가 사용하지만, 29%만 신뢰한다"

→ 사용은 하는데 믿지는 않는다
→ "어쩔 수 없이" 쓰는 상황
→ 더 나은 방법을 원한다
→ Clouvel = 그 "더 나은 방법"
```

### Enforcement Gap

| 도구            | 스펙 기능           | Enforcement |
| --------------- | ------------------- | ----------- |
| GitHub Spec Kit | 템플릿 제공         | ❌          |
| AWS Kiro        | 요구사항 워크플로우 | ❌          |
| ChatPRD         | PRD 생성            | ❌          |
| Notion AI       | 문서 생성           | ❌          |
| Linear          | 이슈 관리           | ❌          |
| **Clouvel**     | 스펙 강제           | ✅          |

**결론: Enforcement 하는 도구 = 0개 → Clouvel 유일**

## 경쟁 분석

### 카테고리별 경쟁자

#### 1. AI IDE (코드 생성)

| 도구              | 특징                         | Clouvel과 관계            |
| ----------------- | ---------------------------- | ------------------------- |
| Cursor            | $29B, 1M+ DAU, 에이전트 모드 | 보완재 (Cursor + Clouvel) |
| Windsurf          | SWE-1 모델, 950 tok/s        | 보완재                    |
| VS Code + Copilot | 20M+ 사용자                  | 보완재                    |

**관계**: Clouvel은 이들을 대체하지 않음. 함께 사용.

#### 2. PM 도구 (스펙 관리)

| 도구         | 특징                     | vs Clouvel             |
| ------------ | ------------------------ | ---------------------- |
| Linear       | $400M+ 밸류, 개발팀 인기 | 비쌈, 이슈 트래킹 중심 |
| Notion       | 범용 문서 도구           | 스펙 특화 아님         |
| Productboard | 엔터프라이즈 PM          | PM 타겟, 개발자 아님   |
| Jira         | 레거시                   | 무거움, 싫어함         |

**관계**: Clouvel은 "개발자를 위한 스펙 도구" 포지션

#### 3. 스펙 도구 (직접 경쟁)

| 도구            | 특징                            | vs Clouvel           |
| --------------- | ------------------------------- | -------------------- |
| GitHub Spec Kit | 템플릿 + 가이드                 | 추천만 함, 강제 없음 |
| AWS Kiro        | VS Code 포크, 구조화 워크플로우 | PM 타겟, MCP 아님    |
| ChatPRD         | MCP PRD 생성기                  | 생성만 함, 강제 없음 |
| OpenSpec        | 스펙 표준 제안                  | 표준만, 도구 아님    |

**관계**: 직접 경쟁이지만 enforcement가 차별점

#### 4. 자율 에이전트 (미래 경쟁)

| 도구                 | 특징            | vs Clouvel       |
| -------------------- | --------------- | ---------------- |
| Devin                | $10B, 자율 코딩 | 스펙 품질에 의존 |
| GitHub Copilot Agent | 백그라운드 PR   | 스펙 품질에 의존 |
| Ramp Inspect         | 30% PR 달성     | 검증 루프가 핵심 |

**관계**: 에이전트가 좋아질수록 스펙이 더 중요 → Clouvel 가치 증가

## 시장 진입 전략

### Phase 1: Cursor 생태계 (2026 Q1-Q2)

```
타겟: Cursor 사용자 (360K+ 유료)
채널: Cursor MCP 디렉토리
메시지: "Cursor로 코딩하기 전에, Clouvel로 스펙 먼저"
목표: 1,000 가입자
```

**왜 Cursor 먼저?**

- Vibe coders가 가장 많음
- MCP 지원 완벽
- 원클릭 설치 가능
- 개발자 커뮤니티 활발

### Phase 2: Claude Code 생태계 (2026 Q2-Q3)

```
타겟: Claude Code 사용자
채널: Claude Desktop 확장, MCP
메시지: "Claude Code + Clouvel = 완벽한 AI 코딩"
목표: 3,000 가입자
```

**왜 Claude Code?**

- Anthropic MCP 창시자
- Claude 사용자 = 기술 수준 높음
- 빠르게 성장 중

### Phase 3: 범용 확장 (2026 Q4+)

```
타겟: 모든 AI 코딩 도구 사용자
채널: VS Code 마켓플레이스, Product Hunt
메시지: "No PRD, No Code"
목표: 10,000 가입자
```

## TAM/SAM/SOM

### TAM (Total Addressable Market)

```
AI 코딩 도구 시장 (2026): $12-20B
```

### SAM (Serviceable Addressable Market)

```
스펙/PRD 관련 도구 시장: ~$500M-1B
- PM 도구의 개발자 세그먼트
- 스펙 생성/관리 도구
- AI 코딩 거버넌스
```

### SOM (Serviceable Obtainable Market)

```
Clouvel 타겟 (5년): $50-100M
- Enforcement 니치 80% 점유
- 개발자 중심 스펙 도구 리더
```

## 시장 타이밍

### 왜 지금인가?

```
1. AI 코딩 폭발 (84% 사용)
   → 스펙 없는 코딩 문제 가시화

2. 신뢰 하락 (29%)
   → 더 나은 방법 필요 인식

3. MCP 표준화 (8M+ 다운로드)
   → 배포 인프라 준비됨

4. "vibe coding hangover"
   → 문제 인식 시작

5. GitHub "intent is the source of truth" 선언
   → 업계 방향성 확인
```

### 타이밍 리스크

```
너무 빠르면?
- 아직 문제 인식 부족
- 대응: 교육 콘텐츠로 문제 인식 유도

너무 느리면?
- 대형 플레이어 진입
- 대응: 빠른 실행 + moat 구축
```
