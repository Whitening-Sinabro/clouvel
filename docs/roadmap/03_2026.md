### 서비스 설명 (상세페이지용)

```
Claude/Cursor로 코딩하는데 "거의 맞는데 뭔가 빠진" 경험 있으시죠?
AI가 요구사항을 멋대로 압축해서 생기는 문제입니다.

Clouvel은 PRD(문서) 없으면 코딩 자체를 차단합니다.
- PRD 없음 → BLOCK (시작 불가)
- PRD 불완전 → WARN (경고 후 진행)
- PRD 완전 → PASS (코딩 시작)

이 서비스는 Clouvel을 **님 환경에 설치하고 동작 확인**까지 해드리는 겁니다.

설치 끝나면 드리는 것:
- 환경별 설정 완료 (CLI/Desktop/VS Code/Cursor)
- PRD 템플릿 (프로젝트에 바로 적용 가능)
- 동작 확인 스크린샷 (차단/통과 시나리오)
- 사용법 가이드 문서

팀이면 DELUXE, 회사면 PREMIUM 추천드립니다.
```

---

### 서비스 제공 절차

```
1. 상담 (당일)
   - 사용 환경 확인 (CLI/Desktop/Cursor/VS Code)
   - 팀 규모, 기존 워크플로우 파악

2. 견적 확정 (당일)
   - 작업 범위 정리
   - 맞으면 결제, 바로 시작

3. 설치 (1~3일)
   - 환경별 MCP 서버 설정
   - PRD 템플릿 적용
   - (DELUXE+) 팀 규칙, Gate 시스템 구성

4. 동작 확인
   - can_code 차단/통과 테스트
   - 스크린샷 + 사용법 문서 전달

5. (DELUXE+) 온보딩 콜
   - 30분 화상으로 사용법 안내
   - 질문 응대

6. 완료
   - 확인되면 구매확정
   - (PREMIUM) 1개월 슬랙 지원 시작
```

# 결제 시스템 구현 가이드

## 개요

결제 시스템은 SaaS의 핵심이지만 실수하기 쉽습니다.
이 가이드는 흔한 실수를 방지합니다.

## 필수 구조

### 1. Webhook 엔드포인트

- [ ] Stripe 이벤트 수신
- [ ] 서명 검증 필수
- [ ] 비동기 처리 (타임아웃 방지)

### 2. Job Queue

- [ ] 재시도 스케줄링 (1/7/14일)
- [ ] 실패 로깅
- [ ] Dead letter queue

### 3. Idempotency

- [ ] 중복 처리 방지
- [ ] 이벤트 ID로 체크
- [ ] DB 트랜잭션

## 흔한 실수

### ❌ 세션 링크 만료 안 고려

문제: 고객에게 보낸 결제 링크가 만료됨
해결: 실시간으로 빌링 포털 링크 생성

### ❌ Webhook 서명 검증 누락

문제: 공격자가 가짜 이벤트 전송 가능
해결: stripe.webhooks.constructEvent() 사용

### ❌ 동기 처리

문제: Webhook 타임아웃 (30초)
해결: 이벤트 받으면 바로 200 응답, 처리는 비동기

### ❌ 재시도 로직 없음

문제: 일시적 실패로 결제 영구 실패
해결: exponential backoff + 최대 재시도 횟수

## 권장 패턴

### ✅ 결제 실패 복구 시퀀스
```

1일 후: 첫 번째 알림 + 자동 재시도
7일 후: 두 번째 알림 + 할인 제안
14일 후: 마지막 알림 + 계정 일시 중지

````

// session.url을 이메일에 포함
