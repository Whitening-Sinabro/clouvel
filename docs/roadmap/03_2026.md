### 구현 가이드라인 예시

#### 결제 시스템 가이드

```markdown
# 결제 시스템 구현 가이드

## 개요

결제 시스템은 SaaS의 핵심이지만 실수하기 쉽습니다.
이 가이드는 흔한 실수를 방지합니다.

## 필수 구조

### 1. Webhook 엔드포인트

- [ ] Stripe 이벤트 수신
- [ ] 서명 검증 필수
- [ ] 비동기 처리 (타임아웃 방지)

### 2. Job Queue

- [ ] 재시도 스케줄링 (1/7/14일)
- [ ] 실패 로깅
- [ ] Dead letter queue

### 3. Idempotency

- [ ] 중복 처리 방지
- [ ] 이벤트 ID로 체크
- [ ] DB 트랜잭션

## 흔한 실수

### ❌ 세션 링크 만료 안 고려

문제: 고객에게 보낸 결제 링크가 만료됨
해결: 실시간으로 빌링 포털 링크 생성

### ❌ Webhook 서명 검증 누락

문제: 공격자가 가짜 이벤트 전송 가능
해결: stripe.webhooks.constructEvent() 사용

### ❌ 동기 처리

문제: Webhook 타임아웃 (30초)
해결: 이벤트 받으면 바로 200 응답, 처리는 비동기

### ❌ 재시도 로직 없음

문제: 일시적 실패로 결제 영구 실패
해결: exponential backoff + 최대 재시도 횟수

## 권장 패턴

### ✅ 결제 실패 복구 시퀀스
```

1일 후: 첫 번째 알림 + 자동 재시도
7일 후: 두 번째 알림 + 할인 제안
14일 후: 마지막 알림 + 계정 일시 중지

````

### ✅ 빌링 포털 실시간 생성
```javascript
// 이메일 발송 시점에 링크 생성
const session = await stripe.billingPortal.sessions.create({
  customer: customerId,
  return_url: 'https://app.example.com/account'
});
// session.url을 이메일에 포함
````

### ✅ 모든 트랜잭션 로깅

```javascript
await logTransaction({
  event_id: event.id,
  type: event.type,
  customer_id: customer.id,
  amount: invoice.amount_paid,
  timestamp: new Date(),
  raw_event: event, // 디버깅용
});
```
