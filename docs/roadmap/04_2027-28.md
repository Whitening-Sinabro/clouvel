유료 (Proprietary):

### H2 (7-12월): 플랫폼 확장

#### 목표

```
다양한 IDE 지원 + Spec-validator 에이전트
→ "어디서든 Clouvel"
```

#### 핵심 작업

| 작업                    | 상세                        | 완료 기준         |
| ----------------------- | --------------------------- | ----------------- |
| JetBrains 통합          | IntelliJ, PyCharm, WebStorm | 플러그인 배포     |
| GitHub Action           | CI/CD 스펙 검증             | 마켓플레이스 등록 |
| Spec-validator 에이전트 | 구현 ↔ 스펙 일치 검증       | 기능 배포         |
| API 공개                | 스펙 검증 API (유료)        | API 런칭          |
| 파트너 프로그램         | 통합 파트너 모집            | 프로그램 런칭     |

#### Spec-validator 에이전트

```
기능:
├── 스펙 ↔ 코드 일치 검증
│   ├── 기능 누락 감지
│   ├── 스펙 외 기능 경고
│   └── API 계약 검증
├── 자동 실행
│   ├── PR 생성 시
│   ├── 커밋 시
│   └── 수동 호출
└── 리포트
    ├── 일치율 점수
    ├── 불일치 상세
    └── 수정 제안

구현 방식:
1. AST 분석으로 코드 구조 추출
2. 스펙과 매핑
3. 누락/초과 기능 식별
4. 리포트 생성
```

#### GitHub Action 예시

```yaml
name: Spec Validation
on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: clouvel/spec-validator@v1
        with:
          spec-path: ./docs/PRD.md
          fail-on-mismatch: true
          report-to-pr: true
```

### H1 (1-6월): 팀/엔터프라이즈 기능

#### 목표

```
SSO, 감사 로그, 승인 워크플로우
→ "엔터프라이즈가 요구하는 모든 것"
```

#### 핵심 작업

| 작업                    | 상세                     | 완료 기준 |
| ----------------------- | ------------------------ | --------- |
| SSO (SAML/OIDC)         | Okta, Azure AD 연동      | 기능 배포 |
| 감사 로그               | 모든 액션 로깅, 내보내기 | 기능 배포 |
| 팀 스펙 승인 워크플로우 | PR처럼 스펙 리뷰         | 기능 배포 |
| 역할 기반 권한          | Admin/Editor/Viewer      | 기능 배포 |
| 팀 분석 대시보드        | 팀 생산성 메트릭         | 기능 배포 |

#### SSO 구현

```
지원 프로토콜:
├── SAML 2.0
│   └── Okta, OneLogin, Azure AD
├── OIDC
│   └── Google, Microsoft, Auth0
└── SCIM 2.0 (H2)
    └── 자동 사용자 프로비저닝

설정 흐름:
1. 관리자가 SSO 설정 접근
2. IdP 선택 (Okta, Azure 등)
3. 메타데이터 교환
4. 도메인 검증
5. 테스트 로그인
6. 강제 적용 (선택)
```

#### 팀 스펙 승인 워크플로우

```
워크플로우:
개발자 → 스펙 작성 → 리뷰 요청
                         ↓
              리드 → 리뷰 → 승인/반려
                         ↓
                      승인 → 코딩 허용
                         ↓
                      반려 → 피드백 → 수정

기능:
├── 리뷰어 지정 (자동/수동)
├── 댓글 + 인라인 피드백
├── 변경 추적
├── 승인 조건 설정
│   ├── 최소 승인 수
│   ├── 필수 리뷰어
│   └── 체크리스트 완료
└── 알림 (이메일, Slack)
```

#### 역할 기반 권한

```
역할:
├── Owner
│   └── 모든 권한 + 결제 + 삭제
├── Admin
│   └── 팀 관리 + 설정 + 멤버 관리
├── Editor
│   └── 스펙 생성/수정 + 템플릿 관리
├── Reviewer
│   └── 스펙 리뷰 + 승인/반려
└── Viewer
    └── 스펙 조회만

커스텀 역할 (Enterprise):
└── 권한 조합으로 커스텀 역할 생성
```

### H2 (7-12월): 엔터프라이즈 영업

#### 목표

```
SOC 2 + 파트너십 + 케이스 스터디
→ "신뢰할 수 있는 엔터프라이즈 벤더"
```

#### 핵심 작업

| 작업              | 상세                            | 완료 기준   |
| ----------------- | ------------------------------- | ----------- |
| SOC 2 Type I      | 보안 감사 시작                  | 인증 획득   |
| 엔터프라이즈 티어 | 커스텀 가격, SLA                | 3개 계약    |
| 파트너십          | Cursor, Claude Code 공식 파트너 | 1개+ 체결   |
| 케이스 스터디     | 성공 사례 3개 발행              | 콘텐츠 완료 |
| 영업 자료         | 피치 덱, 비교표, ROI 계산기     | 자료 완성   |

#### SOC 2 준비

```
범위:
├── Security (필수)
├── Availability (선택)
└── Confidentiality (선택)

준비 사항:
├── 정책 문서화
│   ├── 접근 제어 정책
│   ├── 변경 관리 정책
│   ├── 사고 대응 정책
│   └── 비즈니스 연속성
├── 기술 통제
│   ├── 암호화 (전송/저장)
│   ├── 로깅/모니터링
│   ├── 백업/복구
│   └── 취약점 관리
└── 운영 통제
    ├── 배경 조사
    ├── 보안 교육
    └── 벤더 관리

타임라인:
- 준비: 3개월
- Type I 감사: 1개월
- 리포트 발급: 1개월
- Type II 시작: 6개월 후
```

#### 엔터프라이즈 가격 구조

```
Enterprise 티어:

Base: $50/user/월 (최소 50명)
├── Pro/Team 전체 기능
├── SSO/SCIM
├── 감사 로그 (1년 보관)
├── 우선 지원 (4시간 응답)
├── 전담 CSM
└── 분기별 비즈니스 리뷰

Add-ons:
├── 온프레미스: +$20/user/월
├── 감사 로그 확장 (3년): +$5/user/월
├── 커스텀 통합: 별도 견적
└── 교육/온보딩: $5,000 일회성

Volume 할인:
├── 100-250명: 10% 할인
├── 251-500명: 15% 할인
├── 501-1000명: 20% 할인
└── 1000명+: 별도 협의

예시:
- 100명 팀: $50 × 100 × 0.9 = $4,500/월 ($54K/년)
- 500명 조직: $50 × 500 × 0.85 = $21,250/월 ($255K/년)
```
