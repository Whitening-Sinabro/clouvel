## 고위험 (High Risk)

### 1. AI가 스펙도 완벽히 추론

| 항목     | 내용                                             |
| -------- | ------------------------------------------------ |
| **설명** | AI가 "이런 거 만들어줘"만으로 완벽한 스펙을 추론 |
| **확률** | 중 (3-5년 내 부분 실현)                          |
| **영향** | 높음 (Clouvel 핵심 가치 약화)                    |
| **시기** | 2028-2030                                        |

**조기 경고 신호:**

```
- AI 모델이 스펙 생성 정확도 90%+ 달성
- "스펙 없이 만들어줘"가 표준 워크플로우
- 경쟁 도구들이 스펙 생성 내장
```

**대응 전략:**

```
단기:
- AI 스펙 생성 모니터링
- 스펙 "검증" 가치 강조

중기:
- 스펙 검증/승인 레이어로 피봇
- "AI가 생성한 스펙도 검증 필요" 포지셔닝

장기:
- 인간 승인이 필수인 영역 집중
  - 컴플라이언스 요구사항
  - 엔터프라이즈 거버넌스
  - 법적 책임이 있는 결정
```

**피봇 계획:**

```
현재: "스펙 없이 코딩 금지"
피봇: "AI 스펙도 인간 승인 필수"

새로운 가치:
├── AI 스펙 품질 검증
├── 규정 준수 체크
├── 조직 정책 적용
└── 감사 추적
```

### 2. 대형 플레이어 진입

| 항목     | 내용                                            |
| -------- | ----------------------------------------------- |
| **설명** | GitHub, Cursor, Anthropic이 직접 스펙 기능 추가 |
| **확률** | 높음 (2-3년 내)                                 |
| **영향** | 높음 (시장 빼앗김)                              |
| **시기** | 2027-2028                                       |

**가장 위험한 시나리오:**

```
1. GitHub이 Copilot에 스펙 기능 통합
2. Cursor가 네이티브 스펙 모드 추가
3. Anthropic이 Claude Code에 PRD 강제 기능 추가
```

**조기 경고 신호:**

```
- 경쟁사 로드맵에 스펙 기능 발표
- 인수 제안 (좋은 신호일 수도)
- 유사 기능 베타 출시
```

**대응 전략:**

```
선제적:
- 빠른 moat 구축 (스펙 라이브러리)
- 커뮤니티 충성도 확보
- 오픈소스로 전환 장벽 낮춤

대응적:
- 니치 (enforcement) 유지
- "전문 도구 vs 번들 기능" 차별화
- 파트너십 전환 시도

Exit:
- 인수 대상으로 포지셔닝
- 기술 + 데이터 + 커뮤니티 가치 강조
```

**M&A 시나리오:**

```
GitHub 진입 시:
├── 경쟁하되 인수 대화 오픈
├── "GitHub + Clouvel" 통합 가치 제안
└── 데이터 + 커뮤니티가 핵심 자산

Cursor 진입 시:
├── 파트너십 먼저 제안
├── "Cursor 생태계 내 필수 도구"
└── 통합보다 보완 관계 강조
```

### 3. MCP 표준 변경/폐기

| 항목     | 내용                                    |
| -------- | --------------------------------------- |
| **설명** | Anthropic이 MCP를 크게 변경하거나 폐기  |
| **확률** | 낮음 (Linux Foundation 이전으로 안정화) |
| **영향** | 높음 (핵심 배포 채널 상실)              |
| **시기** | 2027+                                   |

**조기 경고 신호:**

```
- MCP 업데이트 중단
- 주요 플레이어 MCP 지원 축소
- 대체 프로토콜 부상
```

**대응 전략:**

```
예방:
- MCP 커뮤니티 적극 참여
- RFC 기여로 영향력 확보
- 다중 통합 지원

대응:
- IDE 네이티브 플러그인 전환
  - VS Code 확장
  - JetBrains 플러그인
  - 독립 CLI
- 새 표준 빠른 채택
```

- ROI 계산기
- 실패 사례 교육

인센티브:
- 무료 티어 확대

## 피봇 시나리오

### 시나리오 A: AI 스펙 완벽 추론

```
트리거: AI 스펙 생성 정확도 95%+

피봇 방향:
"스펙 생성" → "스펙 검증/승인"

새로운 가치:
├── AI 스펙 품질 검증
├── 인간 승인 워크플로우
├── 컴플라이언스 체크
└── 감사 추적

타겟 시장:
├── 규제 산업 (금융, 의료)
├── 대기업 거버넌스
└── 법적 책임 있는 프로젝트
```

### 시나리오 B: MCP 생태계 축소

```
트리거: MCP 채택 감소, 대체 프로토콜 부상

피봇 방향:
"MCP 네이티브" → "IDE 네이티브"

구현:
├── VS Code 확장 (최우선)
├── JetBrains 플러그인
├── 독립 CLI
└── 웹 앱

장점:
- MCP 의존성 제거
- 더 넓은 시장 접근
```

### 시나리오 C: 엔터프라이즈 채택 저조

```
트리거: 기업 고객 확보 실패

피봇 방향:
"Pro + Enterprise" → "인디 해커 집중"

전략:
├── 가격 인하 ($10/월)
├── 무료 티어 확대
├── 커뮤니티 주도 성장
└── 볼륨 전략

예상 결과:
├── 낮은 ARPU
├── 높은 사용자 수
└── 다른 수익 모델 탐색 (광고, 마켓플레이스)
```

## 리스크 모니터링

### 주간 체크

```
□ 경쟁사 움직임 (발표, 업데이트)
□ MCP 생태계 변화
□ 사용자 피드백 트렌드
□ 핵심 지표 이상 징후
```

### 월간 리뷰

```
□ 리스크 매트릭스 업데이트
□ 조기 경고 신호 체크
□ 대응 전략 유효성 검토
□ 새로운 리스크 식별
```

### 분기별 평가

```
□ 전체 리스크 포트폴리오 리뷰
□ 피봇 시나리오 업데이트
□ 대응 자원 할당 검토
□ 보드/투자자 리포트
```
